## 二、核心架构改造（只加不改旧代码，兼容存量生态）

### 1\. 新增三个关键系统（全局通用，挂在顶层 `BioBase`）

#### ① 特质系统 `TraitSystem`（最关键）

区分三类东西，彻底解耦：

- **枷锁特质（先天限制）**
植物：`无法移动`、`无视觉`、`无听觉`、`无自主意识`、`无法言语`
野兽：`低智本能`、`无法修行`
无机物：`无生命情绪`

- **种族天赋（先天增益）**
树木：光合作用加成、生命力加成
狼：狩猎感知加成

- **变异 / 修行特质（后天破格）**
开灵、化形、灵根、五行亲和、不朽、分身本源、神识……

> 核心规则：
> 所有**禁止性行为**，不靠`if 是不是植物`判断，
> 而是判断：**有没有【枷锁 Trait】**
> 
> 

示例伪逻辑：

```gdscript
# 尝试执行移动
func can_move() -> bool:
	# 只要还挂着「无法移动」枷锁，就禁止
	if trait_system.has_trait("cannot_move"):
		return false
	return true
```

✅ 树没修行：枷锁在 → 不能走
✅ 树修行成精：移除枷锁 → 正常行走

#### ③ AI 等级分层系统（解决「有没有脑子」）

```Plain Text
0级 本能AI：纯状态机，吃喝繁衍，无思考（野草、普通野兽）
1级 灵性AI：基础行为树，有恐惧、趋利，轻度变异生物
2级 智慧AI：完整行为树+语言+情感+社交（妖物、成精草木）
```

- 普通大树：AI=0，只有生长、凋零、光合

✅ 正确：**模板不变，靠特质 \+ 组件动态改造**

❌ Buff 适合临时效果（中毒、流血、短时加成）

- 临时变化 = Buff

- 永久变异 / 种族破格 = Trait 特质

❌ 扩展性为 0，加一个妖物就要改全代码
✅ 正确：判断**枷锁 Trait**，种族只做初始模板
