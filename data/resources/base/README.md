# 资源配置表系统

## 概述

本系统使用 Godot 的 Resource 类创建配置表，解决脚本硬编码问题，使游戏参数可以在编辑器中可视化调整，无需修改代码。

## 已创建的配置表

### 1. 生物配置表 (creature_config.tres)

管理生物的各项属性参数：

- **基本属性**：最大生命值、最大能量值、最大饱食度、移动速度、游荡半径、感知范围、繁殖阈值
- **感知属性**：视觉范围、视觉角度、听觉范围、嗅觉范围
- **消耗速率**：能量消耗速率、饱食度消耗速率、饥饿时能量惩罚、能量耗尽时生命值惩罚
- **繁殖阈值**：能量繁殖阈值

## 使用方法

### 在场景中使用

1. 选中生物节点（例如 Creature 实例）
2. 在 Inspector 面板中找到 "Config" 属性
3. 点击右侧的文件夹图标，选择 `res://01_data/resources/creature_config.tres`
4. 此时生物会使用配置表中的默认值

### 调整参数

有两种方式调整参数：

1. **直接修改配置表**：双击 `creature_config.tres` 文件，在 Inspector 面板中调整各项参数
2. **在实例中覆盖**：在场景中选中生物节点，直接在 Inspector 面板中修改对应属性值

## 优势

1. **可视化编辑**：在编辑器中直接调整参数，无需修改代码
2. **集中管理**：所有生物参数集中在一个配置文件中，便于统一管理
3. **易于扩展**：可以为不同类型的生物创建不同的配置表
4. **运行时可修改**：可以在游戏运行时动态加载不同的配置表

## 扩展建议

1. **创建不同类型的生物配置**：例如 `herbivore_config.tres`、`carnivore_config.tres` 等
2. **添加更多参数**：根据游戏需求添加更多可配置的参数
3. **创建其他类型的配置表**：例如物品配置、技能配置等

## 技术实现

- 使用 `Resource` 类作为配置表的基类
- 通过 `@export` 注解暴露参数到编辑器
- 在脚本中通过 `_load_config()` 方法加载配置值
- 保持向后兼容：即使不设置配置表，也会使用默认值
